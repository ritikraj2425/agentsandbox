# Use Case: LLM Evaluation

Evaluating large language models on coding benchmarks (like HumanEval or SWE-bench) requires actually executing the code generated by the model to see if it passes unit tests. 

Executing AI-generated code directly on your evaluation server is extremely dangerous. An adversarial model could easily generate code that deletes your dataset, modifies your evaluation scripts, or steals your OpenAI API keys.

## The AgentSandbox Solution

AgentSandbox is the industry standard for LLM evaluation.

1. Create a pristine Sandbox for every single evaluation.
2. Inject the model's generated code and the unit tests.
3. Run the unit tests via `agentsandbox run`.
4. The Sandbox captures the exit code, stdout, and stderr perfectly.
5. Destroy the Sandbox.

### Example Flow

```python
import requests

# 1. Start a fresh sandbox
res = requests.post("https://api.agentsandbox.com/v1/sessions", json={"backend": "firecracker"})
session_id = res.json()["session_id"]

# 2. Run the untrusted LLM code
code_to_test = "def add(a, b): return a + b\nprint(add(1, 2))"
action_res = requests.post(f"https://api.agentsandbox.com/v1/sessions/{session_id}/actions", json={
    "command": f"python3 -c '{code_to_test}'"
})

output = action_res.json()["stdout_summary"]
print(f"LLM Output: {output}")

# 3. Destroy it
requests.delete(f"https://api.agentsandbox.com/v1/sessions/{session_id}")
```

Because Firecracker microVMs boot in under 100 milliseconds, you can evaluate thousands of LLM generations in parallel without sacrificing speed or security.
